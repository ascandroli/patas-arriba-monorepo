# Stakeholder Requests

Extracted from stakeholder emails (received in Spanish, documented in English).

---

## 1. User Profile Photos
**Source:** Email 1
**Area:** Client + Server

Users should be able to upload and display a profile photo.

---

## 2. Flip Attendance Logic (Default to "Attending")
**Source:** Email 1
**Area:** Client + Server

Currently, the organizer has to confirm each attendee. The request is to flip this: everyone who signs up is assumed to be attending, and the organizer only marks who did NOT come. The current flow wastes a lot of the organizer's time.

---

## 3. Add +1 / Guest Users to Events
**Source:** Email 1
**Area:** Client + Server

Organizers need to manually add non-platform users ("+1" guests) to an event to track real attendance. Right now, the organizer resorts to an external Excel because the platform doesn't reflect actual headcount. This is a key pain point — the platform data is "not 100% real."

---

## 4. Permission Management (Remove Users from Cars, Events, Platform)
**Source:** Email 1
**Area:** Client + Server

Organizers/admins should be able to:
- Remove someone from a car group
- Remove someone from an event
- Remove/ban someone from the platform entirely

---

## 5. Visual Event Calendar
**Source:** Email 1
**Area:** Client

A calendar view where users can see upcoming events visually (monthly/weekly view).

---

## 6. Revamp Event Categories with Subcategories
**Source:** Email 2
**Area:** Client + Server

The current flat category dropdown is insufficient. The organizer needs hierarchical categories for filtering in exported data (Excel). Proposed structure:

**PROTECTORA (Shelter visits)**
- Viernes Animals Sense Sostre
- Domingo Animals Sense Sostre
- Sábado Badagats
- Domingo Segundas Oportunidades

**EVENTOS (Events)**
- Recogida (collection)
- Mercadillo (flea market)
- Feria (fair)
- Concierto/Fiesta (concert/party)

**OTROS (Other)**

---

## 7. Company/Entity Attendance Tracking
**Source:** Email 2
**Area:** Client + Server

When a company or organization participates in an event, the organizer needs to track:
- Does a company/entity come? (Yes/No dropdown)
- Entity name (free text)
- Number of people (free text)

---

## 8. Glossary / Information Hub
**Source:** Emails 3 & 4
**Area:** Client

A section in the platform where users can look up key terms and concepts (e.g., "SO" = Segundas Oportunidades, "ASS" = Animals Sense Sostre). Proposed as expandable/dropdown format. This has been requested for a long time — new volunteers don't understand the abbreviations.

Should also include fixed reference information visible to all users.

---

## 9. Photo Guide and Merchandising Links
**Source:** Email 3
**Area:** Client

Two pieces of static content to add to the platform:
- **Photo guide:** "How to take photos" — link to an external infographic (not in-platform content)
- **Merchandising:** Link to a document showing products with images, prices, and contact info. Alternatively, an in-platform expandable section if it looks good visually.

---

## Notes

- The stakeholder explicitly acknowledges there is "massive room for improvement" and more requests beyond these.
- Issues #2 and #3 are the highest-impact pain points — they directly affect whether the organizer can trust platform data vs. maintaining a parallel Excel spreadsheet.
- Issues #8 and #9 are related and could be grouped under a single "Information Hub" feature.
